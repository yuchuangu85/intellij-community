Reports when getClass() method is called on a java.lang.Class instance. This is usually a mistake as the result is always equivalent to Class.class. If it's mistake then it's better to remove the getClass() call and to use qualifier directly. If the behavior is intended, then it's better to write Class.class explicitly to avoid confusion.

Example:

  void test(Class<?> clazz) {
    String name = clazz.getClass().getName();
  }

After one of possible quick-fix is applied, the result will be:

  void test(Class<?> clazz) {
    String name = clazz.getName();
  }
