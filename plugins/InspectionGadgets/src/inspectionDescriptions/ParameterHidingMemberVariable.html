Reports method parameters that are named identically to the visible fields of the method's class or surrounding classes if it is a parameter inside of an inner or anonymous class. Such a parameter name may be confusing.

The quick-fix can't be performed in an automatic way, instead it prompts for a new name for a parameter.

Example:

  class Main {
    private final int value;

    public Main(int value) {
        this.value = value;
    }
  }

After the quick-fix is applied:

  class Main {
    private final int value;

    public Main(int input) {
        this.value = input;
    }
  }

Use the Options panel below to configure the inspection:

  1. Enable the first checkbox to ignore parameters of simple setters.
  2. Enable the second checkbox to ignore for private fields in a superclass, which are not visible from the parameter.
  3. Enable the third checkbox to ignore parameters of constructors.
  4. Enable the fourth checkbox to ignore parameters of abstract methods.
  5. Enable the fifth checkbox to ignore parameters of static methods hiding an instance field and to ignore parameters of instance methods in static inner classes hiding an instance field in an outer class. While not strictly hiding, such parameters can still be confusing.
