Reports any calls to Throwable.initCause() where the exception constructor also takes a Throwable cause argument. In this case, the initCause() call can be removed and its argument added to the call to the exception's constructor.

Example:

  try {
      process();
  }
  catch (RuntimeException ex) {
    RuntimeException wrapper = new RuntimeException("Error while processing");
    wrapper.initCause(ex); // Unnecessary call to 'Throwable.initCause()'
    throw wrapper;
  }

A quick-fix is provided to pass the cause to constructor:

  try {
      process();
  }
  catch (RuntimeException ex) {
    RuntimeException wrapper = new RuntimeException("Error while processing", ex);
    throw wrapper;
  }

New in 2016.1
