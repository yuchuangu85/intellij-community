Reports use of generic classes with omitted type parameters. Such raw use of generic types is valid in Java, but it defeats the purpose of type parameters and may mask bugs. This inspection mirrors the rawtypes warning of javac.

Examples:

  •       List list = new ArrayList<String>(); //warning: Raw use of parameterized class 'List' 
      list.add(1);                         //list of strings was created but integer was accepted as well which would lead to runtime exceptions
  • IntFunction<List<?>[]> fun = List[]::new; //no warning as it's impossible to provide type arguments during array creation

Use the checkboxes below to ignore the following cases:

  • raw types are used in objects' construction
  • raw types are used in type casts
  • a type parameter will fail to compile (for example when creating an array or overriding a library method)
  • type parameters are used in parameters of overriding methods
  • a quick-fix is not available
This inspection only reports if the language level of the project or module is 5 or higher
