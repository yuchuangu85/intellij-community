Reports fields explicitly initialized to the same values that the JVM would initialize them to by default.

Example:

  class Foo {
    int foo = 0;
    List bar = null;
  }

After the quick-fix is applied, the result looks like this:

  class Foo {
    int foo;
    List bar;
  }

Use the checkbox below to only report explicit null initialization. For instance, when the checkbox is checked, the above example would be: 

  class Foo {
    int foo = 0;     // no warning
    List bar = null; // redundant field initialization warning
  }
