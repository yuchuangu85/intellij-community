Reports redundant else keywords in if—else statements and statement chains.

An else keyword is redundant when all previous if branches in the chain don't complete normally because they end with return, throw, break, or continue statement. In these cases the statements from the else branch can be placed after the if statement and the else keyword can be removed.

Example:

  if (name == null) {
      throw new IllegalArgumentException();
  } else {
      System.out.println(name);
  }

After the quick-fix is applied the result looks like:

  if (name == null) {
      throw new IllegalArgumentException();
  }
  System.out.println(name);

Disable the checkbox below to ignore cases when the if—else statement is the last statement in the code block.
