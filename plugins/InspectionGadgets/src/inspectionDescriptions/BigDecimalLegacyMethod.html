Reports calls to BigDecimal.divide() or BigDecimal.setScale() which use the integer constants to specify the rounding mode. Since JDK 1.5 methods taking a RoundingMode enum parameter are available and should be prefered instead.

Example:

  new BigDecimal("42").setScale(2, BigDecimal.ROUND_FLOOR);

After the quick-fix is applied, the result looks like this:

  new BigDecimal("42").setScale(2, RoundingMode.FLOOR);
