Reports on any call to java.util.concurrent.locks.Condition.await() not made inside a loop. await() and related methods are normally used to suspend a thread until a condition is signalled as true, and that condition should be checked after the await() returns. A loop is the clearest way to achieve this.

Example:

void acquire(Condition released) { released.await();}

Good code should like this: 

void acquire(Condition released) {
  while (acquired) {
    released.await();
  }
}
