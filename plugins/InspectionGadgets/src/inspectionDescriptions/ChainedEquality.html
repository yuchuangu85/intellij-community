Reports chained equality comparisons (i.e. a == b == c). Such comparisons are confusing: a == b == c means (a == b) == c, but a == b && a == c could be intended.

Example:

  boolean chainedEquality(boolean a, boolean b, boolean c) {
    return a == b == c;
  }

Parentheses can be used to make the comparison less confusing:

  boolean chainedEquality(boolean a, boolean b, boolean c) {
    return (a == b) == c;
  }
