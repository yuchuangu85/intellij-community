Reports non-constant string concatenations used as arguments to SLF4J and Log4j 2 logging methods. Non-constant concatenations will be evaluated at runtime even when the logging message is not logged; this can negatively impact performance. It is recommended to use a parameterized log message instead, which will not be evaluated when logging is disabled.

For example:


  public class Vital {
    private static final Logger LOG = LoggerFactory.getLogger(Vital.class);

    public void saveTheWorld(int i, String s, boolean b) {
      LOG.info("saveTheWorld(" + i + ", " + s + ", "  + b + ")");
      // todo
    }
  }

After the quick fix is applied, the result looks like this:


  public class Vital {
    private static final Logger LOG = LoggerFactory.getLogger(Vital.class);

    public void saveTheWorld(int i, String s, boolean b) {
		  LOG.info("saveTheWorld({}, {}, {})", i, s, b);
      // todo
    }
  }

Use the combobox below to ignore certain higher logging levels. Higher logging levels may be enabled even in production, and the arguments will always be evaluated.
