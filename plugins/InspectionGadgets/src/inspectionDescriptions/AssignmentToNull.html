Reports the assignment of a variable to null, outside of declarations. The main purpose of null in Java is to denote uninitialized reference variable. In rare cases, it's also useful to explicitly assign the variable to null to aid the garbage collection. However using null as a sentinel to denote 'absent value', 'not specified value', 'invalid value' or 'not found element' could be considered as a bad practice and may make the code more prone to NullPointerExceptions. Instead, consider properly defining the sentinel object with the intended semantics, or use library types like Optional to denote the absence of the value.

Example:

  Integer convert(String s) {
    Integer value;
    try {
      value = Integer.parseInt(s);
    } catch (NumberFormatException e) {
      // Warning: null is used to denote an 'invalid value'
      value = null;
    }
    return value;
  }
