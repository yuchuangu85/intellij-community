Reports methods whose return values are never used when called. The return type of such methods can be made void. Quick-fix updates method signature and removes return statements from inside the method.

Example:

  protected String myToUpperCase(String s) { // reported if visibility setting is 'Protected' or higher
    return s.toUpperCase();
  }

  public String setStr(String str) {  // simple setter, reporting depends on setting
    myStr = str;
    return myStr;
  }

  void test() {
    setStr("value");          // return value is unused
    myToUpperCase("result");  // return value is unused
  }

After the quickfix is applied to both methods the result looks like:

    protected void myToUpperCase(String s) {
      // 'return' removed completely as 's.toUpperCase()' has no side effect
    }

    public void setStr(String str) {
      myStr = str;
      // 'return' removed
    }
  ...

NOTE: Some methods might not be reported during in-editor highlighting, due to performance reasons. To see all results, please run the inspection using Analyze | Inspect Code... or Analyze | Run Inspection by Name...

Use second option to control the maximum visibility of methods to be reported.
