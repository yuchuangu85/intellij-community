Reports methods with too many return points. Methods with too many return points may be confusing, and hard to refactor. A return point is either a return statement or the falling through the bottom of a void method or constructor.

Use the field below to specify the maximum number of return points a method is allowed to have.

Use the first checkbox below to ignore guard clauses. Guard clause is an if clause that contains only return statement

Use the second checkbox below to ignore return points inside equals() methods.
