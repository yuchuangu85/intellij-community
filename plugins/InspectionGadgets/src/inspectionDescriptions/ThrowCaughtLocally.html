Reports throw statements whose exceptions are always caught by containing try statements. Using throw statements as a "goto" to change the local flow of control is both confusing and likely to have poor performance.

Example:

  try {
    if (!Files.isDirectory(PROJECTS)) {
        throw new IllegalStateException("Directory not found."); // warning: 'throw' caught by containing 'try' statement
    }
    ...
  }
  catch (Exception e) {
      LOG.error("run failed");
  }

Use the checkbox below to ignore exceptions which are rethrown.
