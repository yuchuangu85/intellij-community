Reports cases when the right side of the assignment is a division that shouldn't be truncated to integer. While occasionally intended, this construction is often buggy.

For example, the following code doesn't change x because of the integer division result: 

  int x = 18;
  x *= 3/2;
So, it should be replaced with: 
  int x = 18;
  x *= 3.0/2;

Use the checkbox below to disable inspection in cases of suspicious but possibly exact divisions e.g., if dividend can't be calculated statically 


  void calc(int d) {
     int x = 18;
     x *= d/2;
  }

New in 2019.2
