Reports arithmetic expressions with too many terms. Such expressions may be hard to understand and bug-prone.

Each primary expression (e.g. parameter or field reference) counts as a term.

Example:

  
    int calc(int a, int b) {
        return a + a + a + b + b + b + b; //Here there are 7 terms and a warning from the inspection. 
    }

Use the field below to specify the maximum number of terms allowed in an arithmetic expression.
