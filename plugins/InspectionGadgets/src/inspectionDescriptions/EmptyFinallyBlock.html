Reports empty finally blocks. Empty finally blocks usually indicate coding errors or could be left over after refactoring the code and can be removed.

This inspection does not report in JSP files.

Example: 

  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {

  }

After the quick fix is applied the result looks like: 

  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
