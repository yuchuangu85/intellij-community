Reports .equals() being called to compare two java.math.BigDecimal numbers. This is normally a mistake, as two java.math.BigDecimals are only equal if they are equal in both value and scale.

Example: if (new BigDecimal("2.0").equals(new BigDecimal("2.00"))) {}

After the quick-fix is applied, the result looks like this:

if (new BigDecimal("2.0").compareTo(new BigDecimal("2.00")) == 0) {}
