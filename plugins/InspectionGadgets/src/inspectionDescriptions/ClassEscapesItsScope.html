Reports usages of classes in a field or method signatures when a class containing in a signature is less visible than the member itself. While legal Java, such members are useless outside of the visibility scope.

Example:

  • public method which returns a private inner class
  • protected field whose type is a package-local class

In Java 9 a module may hide some of its classes by excluding their packages from export. So if the signature of exported API contains a non-exported class, such API is useless outside of the module.

Use the first checkbox below to have this inspection report about the module API which exposes unexported classes.
Note that the option works if the language level of the project or module is 9 or higher.

Use the second checkbox below to have this inspection report about public API which exposes non-accessible classes.

Use the third checkbox below to have this inspection report about package-local API which exposes private classes.
