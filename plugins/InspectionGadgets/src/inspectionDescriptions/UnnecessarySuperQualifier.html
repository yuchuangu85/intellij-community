Reports any unnecessary uses of the super qualifier in method calls and fields references. A super qualifier is unnecessary when the field or method of the super class is not overridden in the calling class.

Example:

  class Foo {
    void foo() {}
  }

  class Bar extends Foo {
    void bar() {
      super.foo();
    }
  }

After the quick-fix is applied, the result looks like this:

  class Foo {
    void foo() {}
  }

  class Bar extends Foo {
    void bar() {
      foo();
    }
  }

Use the checkbox below to ignore qualifiers, which help to distinguish super members access from the same named members of the outer class. See also the following inspections:

  • Java | Visibility | Access of inherited field looks like access of element in surrounding code
  • Java | Visibility | Call to inherited method looks like call to local method
