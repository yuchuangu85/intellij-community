Reports use of <code> tags in Javadoc comments. Since JDK 1.5 these constructs may be replaced with {@code ...} constructs. This allows the use of angle brackets < and > inside the comment, instead of HTML character entities.

Example:

  /**
   * @return empty <code>ArrayList&lt;Integer&gt;</code>
   */
  List<Integer> getList(){ ... }

After the quick-fix is applied, the result looks like this: 

  /**
   * @return empty {@code ArrayList<Integer>}
   */
  List<Integer> getList(){ ... }
