Reports fields whose names are either too short, too long, or do not follow the specified regular expression pattern.

Example: if the inspection is enabled constants and specifies the min field name length 5 (the default) the following constant public static final int MAX = 42; produces a warning because the length of it's name is 3 which is less than 5.

Quick-fix to rename the field is suggested only in the editor.

Use the list below to specify which fields should be checked. When the checkbox for the field type is unchecked, checks for this kind of fields are skipped. For each field type use the fields to specify minimum length, maximum length and regular expression expected for class names. Specify 0 in length fields to skip corresponding checks. Regular expressions are in standard java.util.regex format.
