Reports on any unnecessary uses of this in the code. Using this to disambiguate a code reference may easily become unnecessary via automatic refactorings, and is discouraged by many coding styles.

Example:

  class Foo {
    int x;
    void foo() {
      this.x = 2;
    }
  }

After the quick-fix is applied, the result looks like this:

  class Foo {
    int x;
    void foo() {
      x = 2;
    }
  }

Use the checkbox below to ignore assignments to fields. For instance, this.x = 2; won't be reported, but int y = this.x; will.
