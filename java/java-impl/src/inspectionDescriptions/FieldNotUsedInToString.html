Reports any fields which are not used in the toString() method of a class. This can help discover fields that were added after the toString() method was created and for which the toString() method was not updated.

In the settings accessible in the Generate > toString() dialog it is possible to exclude fields from this check. This inspection will also check for problems with getter methods if the enable getters in code generation option is enabled there.

For example:


  public class Relevant {
    private String name; // not used in toString()
    private int index;
    private int length;

    @Override
    public String toString() {
        return "Relevant{" + "index=" + index +
          ", length=" + length + '}';
    }
  }

A quick fix is provided to generate a new toString() method:


  public class Relevant {
    private String name;
    private int index;
    private int length;

    @Override
    public String toString() {
        return "Relevant{" + "name='" + name + '\'' +
          ", index=" + index + ", length=" + length + '}';
    }
  }
